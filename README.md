  - 併せて、自身のメンタルが不安定な際のセルフコンパッションの役割も兼ねている。

## フロント・バック・DBの構成
- フロント・バックエンドはそれぞれ１か所です。
  
  ### フロントエンド
  - React(vite)を使用
  - メインUIは【 frontend→ src → App.jsx 】に記述
  - 機能毎にトップページに下層ページへの
      リンクを設置しています。

  ### バックエンド
  - サーバーはexpressを使用
  - サーバーとAI APIのロジックはserver.jsに
  
  ### DB（データベース）
  - CMS: 記事データの取得
    ※ 保存先は使用するCMSによって異なります。
  - 記事要約データ: AIにて生成（下記機能欄で述）。
    SQLiteを使用
    保存先は 【 backend → data 】に。

## 機能

  ### AIとのチャット機能
  - claude APIを使用
  - ロジックは【 backend → server.js 】ディレクトリに格納

  ### 記事データの読み込み
  - ロジックはcmsDataLoader.jsに記述
  - 記事データはCMSのテキストデータを使用
  - サーバー起動時に毎回最新の記事内容の初期を行う
    - 目的: APIリクエスト処理と記事情報取得処が被って処理が重くならないようにする為
  <!-- 使用者が作成しやすいようにひな型の作成行う。余 裕がある時でOK -->

  ### AIによる記事データの参照
  - チャットAIへの送信内容・既に保存されている記事データを基に回答を生成

  ### ユーザーの送信内容と関連する記事の検索
  - 後日実装

  
  ### 各記事データの要約を生成
  - 後日実装
  - claude APIを使用
  - CMSの各記事のAPIスキーマIDをUI上で入力後送信。
  そのIDに基づいた記事データのタイトルを取得た上で、
  記事内容を200~300字に要約する機能。
  - 要約データは【 backend → data 】内にデーベースとして保存する
  
  #### 要約内容の編集機能
  - 後日実装
  - 要約したデータは一度フロントエンド上のテストボックスに入り、ユーザーで変更可能。
  - 内容の微調整なら手入力で、全体的に変えた場合は、【再生成】ボタンでも変更可能


## 使用パッケージ

- @anthropic-ai/sdk@0.70.0
  - anthropic(claude)のAPI呼び出し時に使用

- express@5.1.0
  - ローカルサーバー開発時に使用

- cors@2.8.5
  - 別サーバー（フロントエンド）との連携時に使用

- dotenv@17.2.3
  - 環境変数を別ファイルで読み込む際に使用
  - ※ .envファイルに値を記述する際には、自動的に文字列として扱ってくれます。その為、値にクォーテーション不要です。

- axios@1.13.2
  - HTTPリクエスト用パッケージ
  - 以下の目的で使用
    - JSONデータの自動パース
    - エラーハンドリングの一元化
    - ユーザーによってCMSが異なってもパッケージを変えなくても対応可能

- better-sqlite3@12.5.0
  - データベース用パッケージ
  - 以下の目的で使用
    - AIとの過去のチャット履歴を保存・参照
    - CMSの記事内容の要約版を保存・参照
